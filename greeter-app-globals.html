  3. Open up the debugger, put in a breakpoint near the bottom of the <script> tag:
    • What is the scope of appData?
    • What is the scope of all 5 functions?
  4. Wrap the contents of the <script> tag in an IIFE , and open the debugger:
    • Now what is the scope of appData?
    • Now what is the scope of all 5 functions?
    • In the console, test the greetify() and shoutify() functions again - what happens?
  5. Now move the greetify() and shoutify() functions to an external file named textLIB.js:
    • save this file into a folder named src
    • wrap the two functions in an IIFE
    • don't forget to "use strict";
    • to "export" these 2 functions and make them visible outside the IIFE, attach the 2 functions to a global object named textLIB
    • in the HTML file, create a <script> tag that links to textLIB.js
    • in the HTML file, modify the JS calls to greetify() and shoutify() so that they utilize the textLIB object
    • Test the app and be sure that it still works
  6. Now move the rest of the JS code in the HTML file to an external file named index.js:
    • wrap all the code in an IIFE
    • don't forget to "use strict";
    • You DO NOT NEED to create any sort of global object, just keep all of this JS "private" to the IIFE
    • save this file into a folder named src
    • in the HTML file, create a <script> tag that links to index.js
    • test the app and be sure that it still works
  7. Now move the style rules from the HTML file to an external file named default-styles.css:
    • save this file into a folder named styles
    • in the HTML file, create a <link> tag that links to default-styles.css
    • test the app and be sure that it still looks and works the same
